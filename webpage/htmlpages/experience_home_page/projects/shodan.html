Exploring With Shodan

I was curious about network vulnerabilities and how attackers find them. Recently, I wrote an article about the Salt Typhoon hack for my internship (it should be on my LinkedIn in a day or two).

To summarize: Chinese government hackers exploited a backdoor the US government uses to "legally" monitor citizens’ data and messages. With so much sensitive information now at risk, I wanted to dig deeper into how these vulnerabilities are actually discovered and exploited.

That’s when I found Shodan — a search engine for internet-connected devices. It not only shows exposed systems, but also whether they have known vulnerabilities. I decided to write a script that takes a search query, like apache port:80, and uses the Shodan API to see if matching devices have related CVEs (Common Vulnerabilities and Exposures).

At first, I got a little overambitious. I thought about building a whole SQL-like query language for Shodan. But halfway through, I realized: why would anyone use my program when you could just type the query directly on Shodan’s site?

So instead, I shifted focus: my script takes a search query and returns the top 5 most exploited CVEs, organizations, and services associated with the devices.
